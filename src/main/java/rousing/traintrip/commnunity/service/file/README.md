# 이미지 업로드 기능 설명

## 개요
이 기능은 게시판 글에 이미지를 첨부할 수 있도록 구현되었습니다. 파일 업로드는 컴퓨터 환경에 상관없이 동작하도록 설계되었습니다.

## 작동 방식

1. **파일 저장 위치**:
   - 이미지는 각 사용자의 시스템 임시 디렉토리(java.io.tmpdir)에 자동으로 생성된 폴더에 저장됩니다.
   - 이 방식을 통해 환경에 따라 경로가 달라지는 문제를 해결합니다.

2. **접근 URL**:
   - 이미지는 "/images/" 경로를 통해 접근할 수 있습니다.
   - application.yml 파일에서 `file.upload-path` 속성을 통해 URL 경로를 수정할 수 있습니다.

3. **설정 방법**:
   - `application.yml` 파일에서 다음 두 가지 설정을 확인합니다:
     ```yaml
     file:
       upload-dir: upload-images    # 저장될 폴더명 (시스템 임시 디렉토리 내에 생성됨)
       upload-path: /images         # 이미지 접근 URL 경로
     ```

## 주의사항

1. **개발 환경에서는** 이미지가 시스템 임시 디렉토리에 저장되므로 서버를 재시작하면 이미지가 사라질 수 있습니다. 실제 운영 환경에서는 영구적인 저장소로 변경이 필요합니다.

2. **팀원들과 공유할 때는** 각자의 환경에서 서버를 실행하면 자동으로 필요한 디렉토리가 생성되므로 별도 설정이 필요 없습니다.

3. **로컬 개발 시 이미지 확인**:
   - 이미지가 제대로 표시되지 않는 경우, 콘솔에 출력된 디버그 메시지를 확인하여 저장 경로를 확인하세요.
   - 브라우저 개발자 도구를 통해 이미지 URL을 확인하세요.

## 추가 개선점

1. 실제 운영 환경에서는 영구 저장소(AWS S3, Azure Blob Storage 등)로 교체가 필요합니다.
2. 이미지 리사이징, 최적화 기능 추가가 권장됩니다.
3. 이미지 캐싱을 위한 헤더 설정이 추가되면 좋습니다.
